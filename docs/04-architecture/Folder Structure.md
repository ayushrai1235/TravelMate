# TravelMate AI — Folder Structure

**Version:** 1.0.0  
**Date:** 2026-07-03  
**Purpose:** Definitive folder structure for the entire TravelMate AI codebase. Every engineer must follow this structure exactly. Any deviation requires an Architecture Decision Record.

---

## Repository Root

```
travelmate-ai/
├── apps/
│   ├── web/                    # Next.js 16 frontend
│   └── api/                    # FastAPI backend
├── packages/
│   ├── shared-types/           # TypeScript types shared between frontend + BFF
│   └── config/                 # Shared ESLint, TypeScript configs
├── docs/                       # All documentation (this repo)
├── scripts/                    # Dev scripts (seed, migration, etc.)
├── docker-compose.yml          # Local development orchestration
├── docker-compose.prod.yml     # Production-like local testing
├── .github/
│   └── workflows/              # GitHub Actions CI/CD pipelines
├── .env.example                # Environment variable template
├── .gitignore
└── README.md
```

## Key Naming Conventions

| Type | Convention | Example |
|---|---|---|
| React Components | PascalCase | `TripForm.tsx`, `LegCard.tsx` |
| Hooks | camelCase prefixed with `use` | `useTripPlanner.ts` |
| Zustand stores | camelCase suffixed with `Store` | `tripStore.ts` |
| Python files | snake_case | `trip_repository.py` |
| Python classes | PascalCase | `TripRepository` |
| Python functions | snake_case | `get_trip_by_id()` |
| Database tables | snake_case plural | `trip_plans`, `trip_legs` |
| API endpoints | kebab-case plural | `/v1/trip-plans/`, `/v1/trip-legs/` |
| Environment variables | SCREAMING_SNAKE_CASE | `GOOGLE_MAPS_API_KEY` |
| CSS custom properties | kebab-case | `--color-primary-500` |
| Constants | SCREAMING_SNAKE_CASE | `MAX_TRIP_LEGS = 10` |
